# Daisy Docs Platform Plan

## Goal

`docs.daisy.cat` should be the shared documentation home for the Daisy ecosystem rather than a single-product site.

## Product Areas

Current first-class products:

- `DaisyCore`
- `DaisySeries`
- `DaisyConfig`

Potential future products:

- `DaisyData`
- `DaisyEffects`

## Route Model

The docs platform should keep explicit product prefixes:

- `/daisycore/...`
- `/daisyseries/...`
- `/daisyconfig/...`

This keeps:

- URLs predictable
- search results clear
- future product expansion clean

## Site-Level Responsibilities

The docs homepage should answer:

- what Daisy is
- when to choose DaisyCore
- when to choose DaisySeries
- when to choose DaisyConfig
- when to use the full stack together

## Content Standard

Every Daisy docs section should include:

- install path
- first working example
- reference pages
- practical guides
- migration notes where relevant

## Quality Standard

The docs platform should feel:

- premium
- direct
- product-aware
- technically honest
- easy to scan

## End-State Outcome

A new user should be able to land on `docs.daisy.cat`, choose the right product in under a minute, and get to working code without needing to reverse-engineer the Daisy ecosystem from GitHub alone.
