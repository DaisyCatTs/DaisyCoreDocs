<div class="value-strip">

  <div class="value-pill"><strong>DaisySeries</strong><span>The lightweight parser toolkit for materials, sounds, item flags, enchantments, potions, canonical keys, and display names.</span></div>
  <div class="value-pill"><strong>DaisyConfig</strong><span>The typed config layer for YAML loading, reload-safe handles, DaisySeries-aware codecs, and DaisyCore text integration.</span></div>

<div class="doc-shell">
  <div class="doc-lead">
    <span class="eyebrow">Choose Fast</span>
    <h3>Use DaisyCore when you are building plugin systems.</h3>
    <p>DaisyCore is the runtime product. It owns bootstrap, lifecycle, UI systems, text, and the normal Paper plugin flow. If you need commands, menus, sidebars, tablists, and one coherent API style, start there.</p>
    <p><a href="/daisycore/">Open DaisyCore overview</a></p>
  </div>
  <div class="doc-rail">
    <span class="eyebrow">Use DaisySeries</span>
    <h3>When parsing is the problem, not runtime.</h3>
    <p>DaisySeries is the focused utility layer. It removes `valueOf(...)` wrappers, ad hoc alias maps, and hand-written friendly-name helpers without pulling in a full platform.</p>
    <p><a href="/daisyseries/">Open DaisySeries overview</a></p>
  </div>
</div>

<div class="doc-shell">
  <div class="doc-lead">
    <span class="eyebrow">Use DaisyConfig</span>
    <h3>When YAML, reloads, and typed config state are the real problem.</h3>
    <p>DaisyConfig is the typed config layer. It turns config sprawl into explicit codecs, reload-safe handles, DaisySeries-backed value decoding, and DaisyCore-friendly text adapters.</p>
    <p><a href="/daisyconfig/">Open DaisyConfig overview</a></p>
  </div>
  <div class="doc-rail">
    <span class="eyebrow">Use all three</span>
    <h3>For the clean modern Paper stack.</h3>
    <p>Use DaisyCore for runtime systems, DaisySeries for normalized value parsing, and DaisyConfig for typed YAML and reload-safe configuration ownership.</p>
    <p><a href="/daisyconfig/guides/use-daisyconfig-with-daisycore/">See the DaisyConfig + DaisyCore path</a></p>
  </div>
</div>
