## What can you build?

<div class="feature-matrix">
  <div class="feature-card"><strong>Profile runtime flow</strong><p>Use DaisyCore when one feature spans a command, menu, sidebar, and tablist in one runtime-owned flow.</p></div>
  <div class="feature-card"><strong>Config-heavy plugin data</strong><p>Use DaisyConfig when YAML shape, migration safety, module bundles, and shared text ownership become the real maintenance problem.</p></div>
  <div class="feature-card"><strong>Canonical parser layer</strong><p>Use DaisySeries when materials, sounds, flags, enchantments, or effects should stop leaking enum helpers through plugin code.</p></div>
  <div class="feature-card"><strong>Full stack plugin</strong><p>Use all three together when config ownership, canonical values, and runtime systems all matter in the same plugin.</p></div>
</div>
