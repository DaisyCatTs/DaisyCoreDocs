## Discovery contracts

```kotlin
@DaisyCommandSet
object ProfileCommands : DaisyCommandGroup({
    command("profile") {
        player {
            reply("Hello.")
        }
    }
})
```

`DaisyCommandGroup` is the clean default. `DaisyCommandProvider` still exists if you need custom provider logic.

## Availability context

```kotlin
data class DaisyCommandAvailabilityContext(
    val plugin: JavaPlugin,
)
```

`enabled { ... }` runs during DaisyCore command loading. Use it for config-driven startup decisions, not per-execution permission logic.
