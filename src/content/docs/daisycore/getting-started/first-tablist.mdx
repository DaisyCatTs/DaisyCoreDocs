# First Tablist

This page shows the modern DaisyCore tablist path: per-player header and footer sessions with plain strings, MiniMessage strings, `Component` values, or shared config-backed text.

## What this page assumes

<div class="premium-callout note">

- you already have a `DaisyPlatform`
- you enabled `tablists()`
- you want a simple player-facing header and footer
</div>

## Start simple

The easiest path is just `header("...")` and `footer("...")`.

```kotlin
fun profileTablist(player: Player) =
    tablist {
        header("<gradient:#7dd3fc:#c4b5fd>Welcome back</gradient>\n<white>${player.name}</white>")
        footer("<gray>Coins: <white>1,250</white>")
    }
```

Those strings are good when your text is small and local to the feature.

## Step 2: show it to a player

```kotlin
val session = daisy.tablists?.show(player, profileTablist(player))
```

## Step 3: refresh or close it

```kotlin
session?.refreshNow()
session?.close()
```

## Step 4: use shared config/lang text when your plugin already stores header/footer there

```kotlin
fun profileTablist(player: Player) =
    tablist {
        headerLang("tablist.profile.header", viewer = player, "player" to player.name)
        footerLang("tablist.profile.footer", viewer = player, "coins" to "1,250")
    }
```

`tablist.profile.header` is just an example config path. If your plugin keeps all player-facing strings in a lang or config file, this is the clean shared path. If not, the plain-string version is still the right default.

## Components also work

If your plugin is already component-first, the tablist builder also accepts `Component` values directly.

## Current scope

The current public tablist API is intentionally simple:

- header
- footer
- options
- session refresh and cleanup

That is the full honest public contract in this pass.

It does not currently promise richer tab entry formatting or advanced packet-driven customization.
