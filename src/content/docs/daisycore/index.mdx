# DaisyCore

<div class="home-hero">
  <span class="home-kicker">The Full Paper Platform</span>
  <h2>One Kotlin-first runtime platform for real plugin features, not a pile of stitched-together libraries.</h2>
  <p>DaisyCore is for plugins that want one dependency, one platform owner, one shared text model, and one coherent style across commands, menus, sidebars, and tablists.</p>

  <div class="home-actions">
    <a class="home-button" href="/daisycore/getting-started/install/">Install DaisyCore</a>

</div>

<div class="value-strip">
  <div class="value-pill"><strong>One dependency</strong><span>Install DaisyCore once instead of assembling separate command, menu, sidebar, and tablist libraries.</span></div>
  <div class="value-pill"><strong>One runtime owner</strong><span><code>DaisyPlatform</code> owns lifecycle, cleanup, and service startup in one place.</span></div>
  <div class="value-pill"><strong>One text model</strong><span><code>messages(...)</code> powers commands, menus, sidebars, and tablists through the same source.</span></div>
  <div class="value-pill"><strong>One API language</strong><span>The same Kotlin-first style carries through runtime systems instead of changing tone per subsystem.</span></div>
</div>

<div class="doc-shell">
  <div class="doc-lead">
    <span class="eyebrow">What DaisyCore is best at</span>
    <h3>Building one believable Paper feature flow from command to UI.</h3>
    <p>DaisyCore is strongest when one player-facing feature touches multiple systems: a command opens a menu, the menu uses shared text, a sidebar reflects state, and a tablist shows the same player context.</p>
  </div>
  <div class="doc-rail">
    <span class="eyebrow">Current runtime truth</span>
    <ul>
      <li>commands auto-load through runtime discovery</li>
      <li>menus are inline-first by default</li>
      <li>sidebars are keyed and diff-friendly</li>
      <li>tablists are intentionally header/footer-focused</li>
    </ul>
  </div>
</div>

## What DaisyCore includes

<div class="feature-matrix">
  <div class="feature-card"><strong>Commands</strong><p>Auto-loaded command groups, terse executor verbs, async reply helpers, and installed lang/text support.</p></div>
  <div class="feature-card"><strong>Menus</strong><p>Inline-first menu opening, reusable definitions, clean click sugar, and viewer-aware item text.</p></div>
  <div class="feature-card"><strong>Sidebars</strong><p>Keyed lines, focused invalidation, and stable session ownership for player-facing scoreboards.</p></div>
  <div class="feature-card"><strong>Tablists</strong><p>Honest header/footer rendering with session refresh and cleanup behavior.</p></div>
  <div class="feature-card"><strong>Text and placeholders</strong><p>Installed text sources and viewer-aware rendering that keep commands and UI in sync.</p></div>
  <div class="feature-card"><strong>Items</strong><p>Reusable item builders that keep menus and inventory-heavy features cleaner.</p></div>
</div>
