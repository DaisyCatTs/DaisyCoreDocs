# Sidebar DSL

## Main types

- `DaisySidebar`
- `DaisySidebarLine`
- `DaisySidebarRenderContext`
- `DaisySidebarSession`
- `DaisyScoreboardPlatform`

## Main entrypoint

```kotlin
fun sidebar(block: DaisySidebarBuilder.() -> Unit): DaisySidebar
```

## Common builder methods

- `title(Component)`
- `title(String)`
- `titleLang(key, viewer, placeholders...)`
- `title(renderer)`
- `line(key, renderer)`
- `line(key) { text(...) }`
- `line(key) { textLang(...) }`
- `line(key) { visible { ... }; lang(...) }`
- `lineIf(key, visible, renderer)`
- `blank()`
- `options { ... }`

## Important behavior

- line keys should be unique
- blank lines are supported, but DaisyCore still gives them internal unique identity
- sidebars are limited to the normal sidebar line budget
- `invalidate(vararg keys)` is for targeted keyed refreshes
- `invalidateAll()` and `refreshNow()` are the explicit full-refresh paths
- if you install `messages(...)`, sidebar titles and lines can resolve shared lang keys
- viewer-aware PlaceholderAPI rendering follows the same safe behavior as items and menus
