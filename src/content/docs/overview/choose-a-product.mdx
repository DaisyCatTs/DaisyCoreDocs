# Choose a Product

## Choose DaisyCore when you need a platform

Pick DaisyCore if your plugin needs:

- command registration and execution
- menu sessions
- sidebars
- tablists
- shared text and placeholder rendering
- one runtime owner through `DaisyPlatform`

Start here: [DaisyCore install](/daisycore/getting-started/install/)

## Choose DaisySeries when you need lightweight helpers

Pick DaisySeries if your plugin mainly needs:

- enum parsing from config-safe strings
- canonical lowercase serialization keys
- friendly display names
- curated aliases for real plugin inputs

Start here: [DaisySeries install](/daisyseries/getting-started/install/)

## Use both together when your plugin is both UI-heavy and config-heavy

This is the strongest pairing when:

- DaisyCore owns commands, menus, sidebars, or tablists
- DaisySeries parses materials, sounds, or item flags from config

Next step: [Use DaisySeries with DaisyCore](/daisyseries/guides/use-daisyseries-with-daisycore/)
