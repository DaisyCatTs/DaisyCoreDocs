# Materials Dictionary

Use these values for item, icon, block, and config-driven material selection.

## Canonical key rules

- Canonical keys are lowercase underscore names like `diamond_sword`.
- Curated aliases are accepted on input only.
- Namespaced inputs like `minecraft:diamond_sword` are normalized on parse.

## Quick config example

```yaml
icon: acacia_boat
```

## Quick usage example

```kotlin
val value = DaisyMaterials.parse(config.icon)
val key = DaisyMaterials.key(value)
val label = DaisyMaterials.displayName(value)

val typed = required("icon", materialCodec())
```
