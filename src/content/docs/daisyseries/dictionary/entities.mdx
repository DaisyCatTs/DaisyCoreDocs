# Entity Types Dictionary

Use these values for mob settings, entity-based features, and config-driven entity selection.

## Canonical key rules

- Canonical keys are lowercase underscore names like `zombie_villager`.
- Curated aliases are accepted on input only.
- Namespaced inputs like `minecraft:zombie_villager` are normalized on parse.

## Quick config example

```yaml
entity: allay
```

## Quick usage example

```kotlin
val value = DaisyEntities.parse(config.entity)
val key = DaisyEntities.key(value)
val label = DaisyEntities.displayName(value)

val typed = required("entity", entityTypeCodec())
```
