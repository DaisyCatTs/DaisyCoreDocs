# Use DaisySeries with DaisyCore

Use DaisyCore when your plugin needs commands, menus, sidebars, or tablists.

Use DaisySeries when those same flows also need config-heavy values like:

- materials
- sounds
- item flags
- enchantments
- potion effects

```kotlin
val icon = DaisyMaterials.parse(config.icon)
val clickSound = DaisySounds.parse(config.sound)
val enchantment = DaisyEnchantments.parse(config.enchantment)
val effect = DaisyPotions.parse(config.effect)

player.openMenu("Profile", rows = 3) {
    background(Material.GRAY_STAINED_GLASS_PANE) {
        name(" ")
    }
}
```

## Recommended split

- DaisyCore owns runtime, lifecycle, and UI
- DaisySeries owns enum parsing, canonical keys, and friendly names

## Typical real-world flow

- DaisySeries parses config-heavy values like item materials, sounds, enchantments, and potion effects
- DaisyCore handles commands, menus, sidebars, tablists, and shared text rendering
- MiniMessage and components stay in DaisyCore-facing UI code
- config value normalization stays in DaisySeries instead of leaking into feature files
