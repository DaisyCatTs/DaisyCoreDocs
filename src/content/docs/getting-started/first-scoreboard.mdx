# First Scoreboard

This page shows the modern DaisyCore sidebar path: keyed lines, shared text-source rendering, and clean session ownership.

## What this page assumes

<div class="premium-callout note">

- your plugin has a `DaisyPlatform`
- you enabled `messages(...)` and `scoreboards()`
- you want a simple sidebar for one player-facing feature
</div>

## What you'll build

<div class="doc-section">
You will create a small profile sidebar and show it to a player.
</div>

## Step 1: define a sidebar

```kotlin
fun profileSidebar(player: Player) =
    sidebar {
        titleLang("sidebar.profile.title", viewer = player)
        line("name") {
            textLang("sidebar.profile.name", viewer = player, "player" to player.name)
        }
        line("rank") {
            textLang("sidebar.profile.rank", viewer = player)
        }
        line("coins") {
            textLang("sidebar.profile.coins", viewer = player, "coins" to "1,250")
        }
        blank()
        line("online") {
            visible { it.player.isOnline }
            lang("sidebar.profile.online", viewer = player)
        }
    }
```

## Step 2: show it to a player

```kotlin
val session = daisy.scoreboards?.show(player, profileSidebar(player))
```

## Why keys matter

Each line has a key like `rank` or `coins`. DaisyCore uses those keys so a session can refresh specific lines instead of rebuilding everything blindly.

This is why `line(key) { ... }` is the normal path instead of relying on anonymous lines only.

## Step 3: refresh the parts that changed

```kotlin
session?.invalidate("coins")
session?.refreshNow()
```

Use `invalidate("coins")` when one keyed line changed. Use `refreshNow()` when you want a full recompute immediately.

## What the current implementation focuses on

- title rendering
- keyed lines
- lang-based rendering through `messages(...)`
- blank line support
- targeted invalidation
- safe cleanup when the player quits or the platform closes
