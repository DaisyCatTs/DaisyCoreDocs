# Text Bridge

<div class="doc-lead">
  <span class="eyebrow">Reference</span>
  <h3>DaisyConfig stores text. DaisyCore still renders it. The bridge exists to preserve that boundary.</h3>
</div>

<ReferenceTable
  filterable={false}
  rows={[
    { name: 'yamlTextConfigHandle', type: 'object', description: 'Loads a config-backed text source from YAML.', required: true },
    { name: 'asDaisyTextSource', type: 'object', description: 'Exposes the loaded text data in the shape DaisyCore expects.', required: true },
    { name: 'textSource', type: 'object', description: 'Exposes module-backed lang data from DaisyModuleHandle<S>.', required: false },
    { name: 'messages', type: 'object', description: 'Installs the text source into DaisyCore for viewer-aware rendering.', required: true },
    { name: 'textLang', type: 'string', description: 'Resolves shared language keys through DaisyCore once the text source is installed.', required: false },
  ]}
/>

```kotlin
val langHandle = yamlTextConfigHandle(plugin, "lang.yml")

val daisy =
    DaisyPlatform.create(plugin) {
        messages(langHandle.current.asDaisyTextSource())
    }
```

Managed lang files are still data. Placeholder-aware rendering still happens in DaisyCore, not DaisyConfig.
